## v0.4.0 — Dashboard UI
**Released: 2026-03-30**

### Features

- **Dashboard redesign** — Refactored into 4-row layout with dedicated widget components, section headers, and removed Quick Navigation grid.

- **Cost Widget** — Per-agent cost breakdown with period selector (Today / 7 Days / 30 Days). CSS bar chart for historical data, input/output token counts, and USD cost.

- **Agent Metrics** — Tasks completed, average response time, and color-coded success rate per agent. Period toggle for Today and This Week views.

- **System Status** — Real-time overview showing gateway status, active sessions, agents online count, and Telegram connection status.

- **Activity Feed** — Enhanced feed with agent and time range filters, action type icons, agent emoji avatars, and paginated "Load More". Heartbeat events hidden by default.

- **Dynamic model selector** — Chat model picker now loads available models from `openclaw.json` providers instead of hardcoded Claude tiers. Supports any provider (Anthropic, OpenAI, xAI, Google, Mistral).

- **Gateway session viewer** — New "Sessions" tab in Chat sidebar shows live gateway sessions (Telegram, webchat, cron). Click any session to view its conversation history.

### API Endpoints

- `GET /api/usage/history` — Aggregated cost data by day, week, or month per agent
- `GET /api/agents/metrics` — Tasks completed, average duration, success rate per agent
- `GET /api/models` — Available models from all configured providers
- `GET /api/gateway/sessions?key=` — Single session detail with messages

## v0.3.0 — AI-AGIL + Integrations
**Released: 2026-03-30**

### Features

- **Auto-publish on approval** — When a content draft is approved via Telegram or MC, it automatically publishes to LinkedIn. Configurable toggle in Settings > General.

- **User Profile** — New Profile tab in Settings to populate USER.md (name, timezone, context). Syncs to all agent workspaces so every agent knows who they're helping.

- **Board sync** — Content pipeline events automatically create and move cards on the Content Pipeline board. Draft created → card in Ideas. Approved → Review. Published → Published column.

- **Channel viewer** — Office page now shows active channels per agent (Telegram, Gateway, Cron) with recency indicators. Collapsible sessions panel with token usage and costs.

- **Model selector** — Chat UI model picker (Haiku/Sonnet/Opus) now actually controls which Claude model is used via `OPENCLAW_MODEL` environment variable.

- **AI-AGIL methodology** — All 9 agents now have the AI-AGIL skill covering the 6 constitutional principles, task lifecycle, execution modes, wave orchestration, and board integration.

## v0.2.0 — Squad Core
**Released: 2026-03-30**

### Features

- **Squad system** — Create agent teams from templates (Personal, Dev, Hackathon, Social) with one click. Each squad gets a dedicated board, persona files, and skills auto-copied to all agents.

- **Content Pipeline** — Multi-platform content publishing with approval flow. Agents research topics, draft posts, and submit for human approval before publishing.

- **Telegram integration** — Inline approval buttons, media upload, and webhook notifications. Approve or reject agent work directly from Telegram.

- **Memory/RAG system** — Per-agent semantic search with configurable embedding providers (Ollama, OpenAI, Gemini, Voyage, Mistral). Choose RAG or file-based memory per agent via Settings.

- **LinkedIn API** — Full publishing with image upload and article link support. Posts with text, images, and URL previews directly from Mission Control.

- **Skills system** — Global skills (`mission-control.md`, `content-pipeline.md`) copied to every agent. Squad-specific skills (`social-publishing.md`) for specialized agents. Auto-copy on squad creation.

- **Activities system** — Filtering by agent and action type, pagination with offset, DELETE endpoint for cleanup, auto-prune of entries older than 30 days (configurable).

### Improvements

- Chat session persistence — conversations auto-restore when navigating back to Chat
- Memory page loading spinner while fetching data
- Org structure rebuilt from all registered agents on squad create
- Test suite expanded to 70 automated API tests covering all endpoints
- Documentation updated across both repos (README, Architecture, API reference)

### Fixes

- Auth in test suite — gateway token restored after test runs
- Agent ID parsing in memory/RAG CLI output
- `memorySearch` config validation (must be object, not boolean)
- Duplicate cards from repeated squad creation removed
- Board cleanup for test data
